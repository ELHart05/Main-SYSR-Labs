# Lab 02 - Remote Procedure Call (ONC/Sun RPC)

**Module:** Systèmes Répartis - ESI 2CS  
**Language:** C  
**Topics:** ONC RPC, `rpcgen`, XDR serialisation, stub generation

---

## Overview

A distributed calculator service where a **client** calls arithmetic functions (add, subtract, multiply, divide) on a **server** over the network using Sun RPC.  
The service interface is defined in the IDL file `calcul.x`; stubs are generated automatically by `rpcgen`.

---

## Directory Structure

```
lab02-rpc/
├── calcul.x          ← IDL (Interface Definition Language) - source of truth
├── calcul.h          ← generated header (types + function prototypes)
├── calcul_xdr.c      ← generated XDR serialisation routines
├── calcul_clnt.c     ← generated client stub
├── calcul_svc.c      ← generated server dispatch loop
├── calcul_server.c   ← implementation of remote functions  ← EDIT THIS
├── calcul_client.c   ← client main / test code             ← EDIT THIS
└── Makefile
```

> **Rule:** only `calcul_server.c` and `calcul_client.c` are hand-written.  
> Everything else is generated by `rpcgen` and should not be edited manually.

---

## IDL Interface Summary

```c
program CALCUL {
    version VERSION_UN {
        void     CALCUL_NULL       (void) = 0;
        reponse  CALCUL_ADDITION   (data) = 1;
        reponse  CALCUL_SOUSTRACTION(data) = 2;
        reponse  CALCUL_MULTIPLICATION(data) = 3;
        reponse  CALCUL_DIVISION   (data) = 4;
    } = 1;
} = 0x20000001;
```

`data` holds two `unsigned int` operands; `reponse` holds the `unsigned int` result and an `int errno` flag (0 = OK, 1 = overflow / error).

## Building

```bash
make          # builds both 'server' and 'client' binaries
make regen    # re-run rpcgen to regenerate stubs from calcul.x
make clean    # remove all object files and binaries
```

**Requirement:** `libnsl` must be available (`sudo apt install libnsl-dev` on Debian/Ubuntu).

---

## Running

```bash
# Terminal 1 - start the server in the background
./server &

# Verify registration with the portmapper
rpcinfo -p

# Terminal 2 - run the client (pass server hostname)
./client localhost
```

Expected output:

```
Appel de la fonction CALCUL_ADDITION avec les paramètres: 10 et 20
Le résultat de l'addition est: 30

Appel de la fonction CALCUL_ADDITION avec les paramètres: 4294967295 et 1
La fonction distante ne peut faire l'addition à cause d'un overflow
...
```

## Workflow: Adding a New Remote Function

1. Add the function signature to `calcul.x`.
2. Run `make regen` to regenerate stubs.
3. Implement the function body in `calcul_server.c`.
4. Call the function from `calcul_client.c`.
5. Run `make` to rebuild.

## Key Concepts

| Concept | Description |
|---------|-------------|
| IDL (`.x`) | Language-neutral interface specification |
| `rpcgen` | Generates C stubs, XDR encoders, and dispatch code |
| XDR | External Data Representation - platform-independent serialisation |
| Portmapper | Maps program/version numbers to ports (`rpcinfo -p`) |
| Stub | Proxy function on client side that marshals arguments and sends RPC call |
